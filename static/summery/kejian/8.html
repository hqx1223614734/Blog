浮动属性float

  • 浮动定义
    使元素(不完全)脱离文档流,按照指定方向(左右)发生移动,遇到父级边界或者相邻的浮动元素就停下来.
  • 文档流定义
    文档中可以显示对象在排列时所占的位置
    (简单说就是从头到尾按照文档的顺序排列,该在什么位置就在什么位置)
  • 脱离文档流
    在页面中不占位置

一.浮动的一般情况

  • 浮动第一种情况
    把框一 左/右 浮动,当它碰到左右某一个边框就停止

<head>

    <style>

        li.box1{

        float:left

    }

    </style>

</head>

<body>

    <ul>

        <li class='box1'>框1</li>

        <li>框2</li>

        <li>框3</li>

    </ul>

</body>
  • 浮动第二种情况
    同时浮动,后一个碰到前一个框就停止

<head>

    <style>

        li{

        float:left

    }

    </style>

</head>

<body>

    <ul>

        <li>框1</li>

        <li>框2</li>

        <li>框3</li>

    </ul>

</body>
  • 浮动第三种情况(问题)
    如果包含框太窄，无法容纳水平排列的三个浮动元素，那么其它浮动块向下移动，直到有足够的空间。
  • 浮动第四种情况(问题)
    当包含框太窄,浮动元素往下掉时.并且浮动元素的高度不同，那么当它们向下移动时可能被其它浮动元素”卡住”

二.浮动的影响,及清除浮动

  • 给父级定义高度
    子元素浮动后,父级无法获取高度,需要手动添加
    优点:简单,代码少,容易掌控
    缺点:只适合高度固定的布局,要给出精确的高度,如果高度和父级div不一样时,会产生问题.对于响应式布局会很大影响
  • 浮动元素最后一个加上一个空标签,写入样式clear:both;
    原理:给这个空元素清除直接清除浮动,它就会位于浮动元素的下发,就会达到像文档流的布局效果.而我们这个元素没有设置宽高,所以父级元素的高度就是基于此元素的上边宽计算的
    优点:简单,代码少,浏览器支持好,不容易出现奇怪问题
    缺点:初学者不能理解,浮动布局太多,增加空标签太多,感觉不好
  • 给父级定义伪类:after和zoom
    优点:浏览器支持好,不容易出现怪问题
    缺点:代码多,不少初学者不理解
.clearfix:after{content:'';display:block;clear:both;}

.clearfix{zoom:1;}
  • 给父级定义overflow:hidden;
    优点:简单,代码少,浏览器支持
    缺点:不能和position配合使用,超出的部分会被影藏
  • 父级定义overflow:auto;
    优点:简单,代码少,支持浏览器
    缺点:内部高度超过父级时会出现滚动条

三.浮动和inline-block的对比

  • inline-block会出现间隙
    第一个就是我们的inline-block
    第二个就是我们的float

  • 面对不同的高度对齐方式不同
    第一个是我们的inline-block
    第二个是我们的float
